# StudySync – Product Backlog & Sprint Plan

## Project Overview
**App:** StudySync  
**Purpose:** A centralized platform for college students to manage tasks, collaborate with peers, and organize study schedules efficiently.  
**Team Members:** Robyn, Desmond, Rashard

---

## Product Backlog

| # | User Story | Priority |
|---|-----------|----------|
| 1 | As a student, I want to register an account so that I can access StudySync. | High |
| 2 | As a student, I want to log in securely so that my data is protected. | High |
| 3 | As a student, I want to create tasks with deadlines so that I can manage my assignments. | High |
| 4 | As a student, I want to view all my tasks on a dashboard so that I can see what's due. | High |
| 5 | As a student, I want to create a study group so that I can collaborate with classmates. | High |
| 6 | As a student, I want to join an existing study group so that I can work with my peers. | Medium |
| 7 | As a student, I want to send messages within a study group so that we can communicate easily. | Medium |
| 8 | As a student, I want my deadlines synced to Google Calendar so that I get reminders automatically. | Medium |
| 9 | As a student, I want to update or delete my tasks so that my list stays current. | Medium |
| 10 | As a student, I want to see all members of my study group so that I know who's in my team. | Low |

---

**Sprint Goal:** Establish core backend structure for user management, task management, and study group creation.

## Notes
- Sprint items #1 and #2 align with the **User model** and authentication logic.
- Sprint items #3 and #9 align with the **Task model** and CRUD operations.
- Sprint item #5 aligns with the **StudyGroup model** and group management logic.
- Items #6–#10 are candidates for Sprint 2 once the backend foundation is established.
